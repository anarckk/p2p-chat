删除无意义的历史用户提示词。

无意义的标准：
1、简单指令（如"修复"、"继续"、"是"）
2、单纯 Skill 调用相关的 prompt
3、元指令（如 update-require、get-all-user-prompts、import_e2e）
4、"更新记忆"指令
5、琐碎/不完整的文件（如 1.md、B.md、实现.md）
6、错误信息反馈（如 'nul' 不是内部或外部命令）
其他你觉得是无意义的，但是想删，就要询问用户。

- 你要从内容中分析文件是否无意义。类似`不需要管webkit，只需要chrome测试即可。`这是非常有意义的。
- 即使是 ide_selection/ide_opened_file 开头的文件，如果后面跟着用户的具体输入内容（如需求、bug 反馈等），也是有意义的。
- 提示词中用户输入的中文文字数量较多的，大概率是有意义的。
- <ide_opened_file> </ide_opened_file> 里的内容是无意义的。
- "你要自动维护和更新记忆" 这类指令不能删。
